# Release v1.1.7

**Release Date:** 2026-06-13

## Robust Token Auto-Refresh

### Problem

When the stored access token had expired and the in-memory refresh token was
stale, every tool call failed with a cryptic error:

```
Error: Token refresh failed: {'status': 503, 'body': {}, 'error': 'Invalid Params: invalid refresh_token'}
```

Two gaps caused this:

1. **The server never re-read tokens from `.env` at runtime.** After a fresh
   re-authorization (`generate_tokens.py` writing new tokens to `.env`), the
   running server kept using its old in-memory tokens and kept failing until it
   was fully reconnected/restarted.
2. **A dead refresh token produced an opaque error** that didn't tell the user
   what to do.

### Fix

- **`_reload_tokens_from_env()`** (new): re-reads `WITHINGS_ACCESS_TOKEN` /
  `WITHINGS_REFRESH_TOKEN` from `.env` at runtime.
- **`refresh_access_token()`** now re-reads `.env` *before* refreshing, so a
  token rotated by another process or written by a re-authorization is picked up
  **without restarting the server** — the freshest stored value always wins.
- A 401 from the API still triggers a single refresh-and-retry
  (`_make_request`), and that refresh now self-heals from `.env`.
- When the refresh token is genuinely invalid/expired, the server raises a
  **clear, actionable message** instead of the raw API payload:

  ```
  Refresh token invalid or expired — re-authorization required.
  Run `python generate_tokens.py` (or use the get_authorization_url flow) to issue new tokens.
  ```

- **`ensure_valid_token()`** reloads from `.env` if no access token is held,
  before giving up.

### Net effect

- After a re-auth, the next tool call self-heals — no reconnect needed.
- Expired access tokens refresh transparently as long as the refresh token is alive.
- A dead refresh token now tells you exactly how to fix it.

### Changes

Modified `src/withings_mcp_server/auth.py`:

- Added `_reload_tokens_from_env()`
- `refresh_access_token()`: reload-before-refresh + clear re-auth error on invalid token
- `ensure_valid_token()`: reload from `.env` before failing on missing token
